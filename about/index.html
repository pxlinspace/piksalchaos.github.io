* Tools *

Art+Animation

  • Adobe Animate: My go-to animation software, though it honestly feels like it's gotten a bit more buggy and annoying to use.
  • Krita: Super high quality for a free and open source drawing software!
  • Procreate: Great for when I'm on the go or for quick concept sketches!
  • Aseprite: It's a joy to make sprites with this program! By default it's payed, but you can get it for free if you compile the source code .
  • Pixelorama: Another open source pixel art software. This was what I used when compiling Aseprite was too hard. It's cool in that you can bind different tools to either mouse button!
  • Flipaclip: Very fun for making quick animation sketches when I'm bored.
  • Tahoma2D: It's just the Opentoonz animation program but more simplistic! The vertical timeline makes me feel more professional than I actually am.

Languages I know from best to worse

  • GDScript: It's the language of the Godot game engine! The syntax is very similar to Python, with a few differences that make it more integrated with the engine.
  • Python: It's basically the easiest programming language lol, though I put it below GDScript since I don't really have that many use cases for it right now.
  • Lua: Unlike everyone else who learns Lua for Roblox scripting, I learned it for the LÖVE framework. It's fun, but it's differences make it feel a little weird and annoying to use.
  • JavaScript+TypeScript: Ehhh it's okay. Very flexible, but I find web development to be more of a hassle than game development.
  • C/C++: This is what we used back in my old high school Robotics team. Haven't had much need to use it since then, so it's slowly phasing out of memory.
  • Java: I never took AP CSA as a class, but I signed up for the AP exam. I literally only studied the syntax the night before the exam, and then proceeded to pass with a perfect score of 5. I even had enough extra time to make a full drawing of Hatsune Miku on the scratch paper which they then proceeded to take away :(
  • C#: My first game engine was actually Monogame (based on the XNA framework). I was following in the footsteps of my idol ConcernedApe, but gave up on it because it was too hard at the time :(
